# 숨표 베타 QA 체크리스트
작성일: `2026-04-17`

## 1) 기능 체크
- 앱 실행 후 Home/Result/Routine/Report 탭 전환이 정상 동작한다.
- `Analyze now` 호출 시 로딩 UI가 노출되고 결과 화면으로 이동한다.
- 잘못된 입력(빈 transcript) 시 에러 안내가 뜬다.
- API 서버 미실행 상태에서 네트워크 에러 안내가 뜬다.
- `Start voice input`으로 음성 인식 시작/중지가 가능하다.
- 음성 인식 결과가 텍스트 입력창에 반영된다.
- 결과 화면에서 태그/점수/confidence/hold_decision 표시가 정상이다.
- `GET /checkins`에 최근 분석 결과가 저장되어 조회된다.

## 2) 안전/정책 체크
- 앱 내 “의료 진단 아님” 문구가 온보딩/결과 화면에 포함되어 있는지 확인한다.
- 위기 키워드 대응 문구/연결 동작이 설계대로 노출되는지 확인한다.
- 민감정보(원문 transcript) 저장 정책이 문서와 일치하는지 확인한다.

## 3) 성능/안정성 체크
- 연속 10회 분석 요청 시 앱 크래시가 없어야 한다.
- API 응답 헤더 `x-request-id`, `x-process-time-ms`가 포함되어야 한다.
- 서버 로그에 요청 경로/상태코드/처리시간이 남아야 한다.

## 4) 베타 릴리즈 전 체크
- `python -m unittest discover -s tests -p "test_*.py"` 통과
- `flutter analyze` 통과
- `flutter test` 통과
- Android 실제 기기 1대 이상에서 음성 인식 권한 플로우 점검
- iOS 시뮬레이터 또는 실제 기기에서 마이크/음성 인식 권한 점검

## 5) 베타 운영 체크
- 일일 KPI: 분석 성공률, API 에러율, D1 리텐션
- 주간 KPI: 주 3회 이상 체크인 비율, 루틴 진입률
- 사용자 피드백 채널: 설문 링크/오픈채팅/이메일 중 1개 이상 고정
