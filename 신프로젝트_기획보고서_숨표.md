# 신프로젝트 기획 보고서
프로젝트명(가칭): `숨표 (Sumpyo)`  
작성일: `2026-04-17`

## 1. 프로젝트 개요
`숨표`는 사용자의 짧은 음성 체크인을 기반으로 감정/스트레스 상태를 분석하고, 맞춤형 회복 루틴을 제안하는 감정 회복 코치 앱이다.  
기존 팀 역량인 녹음/녹취(STT)와 데이터 모니터링 경험을 재사용해 빠른 MVP 출시를 목표로 한다.

## 2. 추진 배경
- 기존 명상/저널링 앱은 입력 부담이 커서 이탈이 빠르다.
- 사용자는 힘든 상태를 느껴도 원인을 구조적으로 파악하기 어렵다.
- 음성 입력은 텍스트 입력 대비 마찰이 낮아 습관화에 유리하다.

## 3. 핵심 가치 제안
- 하루 3분 체크인으로 현재 상태를 점수와 태그로 시각화한다.
- 타인 평균이 아닌 개인 기준선 대비 변화를 중심으로 분석한다.
- 바로 실행 가능한 2~5분 회복 루틴을 추천한다.

## 4. 타깃 사용자
- 25~40세 지식노동자(개발자, 기획자, 직장인)
- 감정 기록 습관은 약하지만 자기관리 니즈는 높은 사용자
- 업무 스트레스와 번아웃 전조를 조기에 관리하려는 사용자

## 5. 상태분석 방식
1. 입력 수집: 음성(60~120초), STT 텍스트, 시간대 메타데이터
2. 텍스트 분석(주 신호): 불안/피로/압박/무기력/안정 멀티라벨 분류
3. 음성 분석(보조 신호): 말속도, 침묵 비율, 에너지와 피치 변동
4. 스코어링: 텍스트+음성 가중 결합으로 회복 점수(0~100) 산출
5. 개인화: 첫 7~14일 기준선 학습 후 개인 변화량 중심 반영
6. 결과 출력: 점수, 상위 태그 2~3개, 근거 설명, 추천 루틴 1~2개

## 6. MVP 범위(4주)
- 음성 체크인 녹음
- 자동 녹취(STT)
- 감정 태그와 회복 점수 제공
- 마이크로 루틴 추천
- 리마인드와 7일 리포트

MVP 제외 범위:
- 의료 진단/치료 기능
- 커뮤니티 기능
- 웨어러블 연동
- 실시간 스트리밍 분석

## 7. Pros and Cons
### Pros
- 기존 기술 자산 재사용률이 높아 개발 속도가 빠르다.
- 음성 중심 UX로 초기 온보딩 장벽이 낮다.
- 개인화 리포트 기반 구독 모델 설계가 용이하다.
- 수면/집중/번아웃 예방 등으로 확장성이 높다.

### Cons
- 의료/진단 앱으로 오해될 리스크가 있다.
- 감정 분석 오탐 시 신뢰도가 빠르게 하락할 수 있다.
- 음성/감정 데이터 보호에 대한 운영 부담이 크다.
- 웰니스 앱 특성상 리텐션 관리 난도가 높다.
- 유사 대체재가 많아 메시지 차별화가 필요하다.

## 8. 핵심 KPI(초기)
- D7 리텐션: 30% 이상
- 주 3회 이상 체크인 비율: 40% 이상
- 루틴 실행률: 35% 이상
- 2주 후 자기인식 도움 긍정 응답: 60% 이상

## 9. 리스크 및 대응
- 의료 오해 리스크: 전 구간에 웰니스 도구(진단 아님) 문구 명시
- 정확도 리스크: 결과 수정 피드백 루프로 모델 보정
- 개인정보 리스크: 암호화 저장, 보관 기간 정책, 삭제권 제공
- 위기상황 리스크: 자해/위기 키워드 탐지 시 즉시 도움 리소스 안내

## 10. 4주 실행 계획
1. 1주차: 문제 인터뷰(8~10명), 태그 체계/점수 로직 v1, 와이어프레임 확정
2. 2주차: 녹음/업로드/STT 파이프라인 구현, 분석 API 연동
3. 3주차: 규칙 기반 루틴 추천, 리마인드/주간 리포트 구현
4. 4주차: 클로즈드 베타(30명), 이탈 구간 분석, 유료 전환 실험

## 11. 결론
`숨표`는 팀의 기존 기술 역량을 가장 효율적으로 재사용하면서 새로운 웰니스 도메인으로 확장할 수 있는 프로젝트다.  
초기 성공의 핵심은 절대적 정확도보다 신뢰 가능한 설명, 안전한 데이터 처리, 습관화 가능한 UX 설계에 있다.
