# 상태분석 작동 플로우
작성일: `2026-04-17`

## 1) 플로우차트 (Mermaid)
```mermaid
flowchart TD
    A[사용자 음성 체크인 시작] --> B[음성 녹음 60~120초]
    B --> C[STT 변환]
    C --> D[자가보고 입력<br/>스트레스 1~5]
    D --> E[텍스트 감정 신호 분석<br/>불안/피로/압박/무기력/안정]
    E --> F[개인 기준선 조회<br/>최근 7~14일]
    F --> G{기준선/데이터 충분?}

    G -- 아니오 --> H[학습기간 모드<br/>점수 신뢰도 낮음 표시]
    H --> I[보수적 피드백 + 짧은 루틴 추천]
    I --> Z[결과 저장]

    G -- 예 --> J[변화추세 계산<br/>7일 대비 악화/개선]
    J --> K[음성 보조신호 계산<br/>말속도/침묵비율/에너지]
    K --> L[위험점수 계산<br/>자기보고50 + 텍스트35 + 추세10 + 음성5]
    L --> M[회복점수 = 100 - 위험점수]
    M --> N{신뢰도 임계치 통과?}

    N -- 아니오 --> O[판단보류 메시지<br/>추가 체크인 유도]
    O --> Z

    N -- 예 --> P[결과 출력<br/>회복점수 + 상태태그 2~3개 + 근거]
    P --> Q[맞춤 루틴 1~2개 추천]
    Q --> R{위기 키워드 감지?}
    R -- 예 --> S[일반결과보다<br/>도움 리소스 우선 노출]
    R -- 아니오 --> T[일반 코칭 메시지 노출]
    S --> Z
    T --> Z

    Z --> U[사용자 피드백 수집<br/>맞아요/아니에요]
    U --> V[개인화 가중치 업데이트]
```

## 2) 텍스트 단계 요약
1. 사용자가 60~120초 음성 체크인을 녹음한다.
2. 음성을 STT로 텍스트 변환하고, 스트레스 자기보고(1~5)를 함께 받는다.
3. 텍스트 감정 신호를 주신호로 분석하고, 음성 특성은 보조신호로만 사용한다.
4. 개인 기준선(최근 7~14일)과 비교해 오늘의 변화량을 계산한다.
5. 위험점수(`자기보고50 + 텍스트35 + 추세10 + 음성5`)를 계산한다.
6. 회복점수(`100 - 위험점수`)를 산출한다.
7. 신뢰도 임계치를 통과하지 못하면 판단보류 메시지를 출력한다.
8. 통과하면 점수/태그/근거와 맞춤 루틴을 제공한다.
9. 위기 키워드가 감지되면 일반 결과보다 도움 리소스를 우선 노출한다.
10. 사용자 피드백(맞아요/아니에요)로 개인화 가중치를 업데이트한다.
